# 热度轮转策略定义
**创建时间**: 20260320

## 策略概述

基于东财人气排名（popularity_rank）均值回归特性的单仓位/多仓位全市场轮转策略。

利用热度排名的均值回归特性：当一只股票的热度跌到近期最低点时买入，
等热度回升后卖出，然后立即从全市场选出下一只"最冷门"的股票。
资金始终满仓运转，没有空窗期。

## 核心逻辑

### 数据来源

- **热度排名**：`my_trend.popularity_rank`（port 3310，每日17:00更新）
  - rank=1 为最热门，rank 越大越冷门（全市场约 5,490 只股票）
- **行情数据**：`my_stock.market_daily` + `adj_factor`（前复权）

### 核心指标

**heat_position**（热度相对位置）：
```
heat_position = (当前rank - N日最低rank) / (N日最高rank - N日最低rank)
```
- 值接近 1.0 → rank 处于近期最高 → **热度处于近期最低** → 买入机会
- 值接近 0.0 → rank 处于近期最低 → **热度处于近期最高** → 卖出时机

**rank_surge**（排名骤升倍数）：
```
rank_surge = 当前rank / N日均值rank
```
- 值越大 → 该股相对自身常态"冷"了越多 → 选股时优先
- 例如：平时 rank=100 的股票，当前 rank=300 → surge=3.0（冷了3倍）

### 交易规则

**买入条件（信号日收盘后确定，T+1收盘执行）**：
1. 当前无持仓（或有空仓位）
2. heat_position >= buy_threshold（默认 0.80）
3. 日成交额 >= min_deal_amount（默认 5000万）
4. 次日非停牌
5. 在候选中选 rank_surge 最大的（均值回归弹性最大）

**卖出条件（信号日收盘后确定，T+1收盘执行）**：
1. heat_position <= sell_threshold（默认 0.20，热度已回升）
2. 或持仓天数 >= max_hold_days（默认 9999 = 无超时）

**轮转**：卖出信号和买入信号同日产生，T+1 同时执行（资金几乎无空窗期）

## 多仓位扩展（n_positions >= 2）

- 维护最多 n_positions 个持仓槽位，资金均分
- 有空仓时才买入，按 rank_surge 从高到低填满
- 任一仓位 heat_position <= sell_threshold 触发卖出该仓位
- 卖出当日立即补仓（同日信号，次日执行）
- 期望：降低集中度风险，提升稳定性，但可能降低绝对收益

## 策略优势

1. **逻辑简单清晰**：基于热度均值回归，不依赖基本面复杂分析
2. **选股范围广**：全市场扫描 5490 只股票，随时找到最冷门标的
3. **资金利用率高**：满仓轮转，无空窗期
4. **信号频率合适**：平均每次持仓约 8-12 天，不过度频繁

## 策略局限

1. 不考虑交易成本（双边手续费约 0.2%，每年约 20-30% 的摩擦成本）
2. 流动性风险：大资金可能影响成交
3. 模型对极端行情（熔断、连续跌停）处理简化
4. 单一信号源，无基本面过滤
