# 全市场批量计算模式

全市场股票指标计算（~5500只 × 多周期 × 多复权）的标准模式，MA 研究中验证通过。

## 问题

逐只股票串行读写 → 5500 次 SELECT + 5500 次 INSERT，耗时数小时。

## 解决方案：批量 I/O + 向量化计算 + 多复权并行

### 核心三层优化

1. **批量读取**：每批 100 只股票一条 SQL，5500 次 → 55 次
2. **向量化计算**：groupby('ts_code') + rolling，替代逐只 calc
3. **多复权并行**：bfq/qfq/hfq 三种复权用 ProcessPoolExecutor 并行

### 内存控制

| 每批 | 峰值内存 | 适用场景 |
|------|---------|---------|
| 50 只 | ~211 MB | 云上 2G 实例 |
| 100 只 | ~422 MB | 云上 4G 实例（默认） |
| 200 只 | ~844 MB | 本地 / 8G+ 实例 |

### 写入优化

- 分片 to_dict（每 1000 行）+ batch_upsert，连接复用
- 禁止全量 to_dict（100只 → 50GB 内存爆炸）

### 性能对比

| 场景 | 旧方案 | 新方案 (串行) | 新方案 (3并行) |
|------|--------|-------------|--------------|
| 日线 × 1 adj | ~4h | ~10-15min | — |
| 日线 × 3 adj | ~12h | ~30-45min | ~10-15min |
| 全周期 × 3 adj | ~15h+ | ~40-60min | ~15-20min |

### CLI 参数

```bash
python compute_stock_ma.py                              # 日线 bfq（默认）
python compute_stock_ma.py --all-adjs --workers 3       # 3种复权并行
python compute_stock_ma.py --batch-size 50 --workers 2  # 自定义
python compute_stock_ma.py --codes 000001.SZ,600519.SH  # 指定股票
```

## 复用指南

新指标的全市场计算参考此模式：
1. `load_*_batch(codes, freq, adj, conn)` — 批量读取
2. `calc_*_batch(df)` — groupby + 向量化
3. `write_batch(df, model, conn)` — 分片写入
4. `compute_one_adj(freq, adj, batch_size)` — 单复权流程
5. 主入口调度并行（workers > 1 时 ProcessPoolExecutor）
