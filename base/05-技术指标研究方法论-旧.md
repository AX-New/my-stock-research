# 技术指标研究方法论

> 从 MACD/MA/RSI/换手率 研究中提炼的通用框架，适用于任意技术指标。
> 核心思路：分层验证 + 信号后续收益统计 + 噪音过滤

---

## 一、研究原则

### 1.1 基本信条

- **从顶底出发，反向研究指标特征**：先确认历史顶底，再观察指标在那些时刻呈现什么特征
- **假设所有信号都是噪音，用数据证伪**：不预设某信号有效，用统计结果说话
- **分层验证，逐步收窄**：从大盘到行业到个股，每一层都必须通过才推进

### 1.2 评估标准

| 维度 | 含义 | 门槛 |
|------|------|------|
| **胜率** | 信号发出后，价格朝预期方向运动的概率 | 指数级: >65% 有意义，>80% 强信号；个股级: >55% 有意义（因20-30pp衰减） |
| **收益率** | 信号发出后的平均收益/亏损幅度 | 需要覆盖交易成本 |
| **频率** | 信号出现的次数 | 太稀有（<1次/年）无法依赖 |

三者缺一不可。

### 1.3 牛熊周期是所有分析的前提（强制）

**任何技术指标信号的分析，都必须按牛熊阶段分别统计。**

同一信号在牛市和熊市下表现完全相反（如金叉在牛市胜率89%，熊市仅33%）。不分牛熊的汇总统计会掩盖真实规律。

**前置阅读**：
- `report/01-a-share-bull-bear-cycles.md` — 21个牛熊阶段划分
- `report/04-index-macd-methodology.md` — 信号在牛熊中的差异表现

**执行标准**：
- 每个信号输出三行统计：**全量 / 牛市 / 熊市**
- 结论判定以牛熊分组为准，全量仅供参考
- 牛熊中表现相反的信号是**条件性有效信号**，不是噪音

**牛熊子类型**：

| 子类型 | 代表区间 | 特征 |
|--------|---------|------|
| **大牛市** | 998→6124, 1850→5178 | 全面上涨，24~28月，涨幅180%~514% |
| **反弹牛** | 1665→3478 | 政策驱动，10月，涨幅109% |
| **慢牛/结构牛** | 2638→3587, 2441→3732 | 结构分化，24~25月，涨幅36%~53% |
| **急跌熊** | 6124→1665, 5178→2638 | 流动性危机，7~12月，跌幅49%~73% |
| **阴跌熊** | 3478→1850, 3732→2635 | 经济下行，36~46月，跌幅29%~47% |
| **中等熊** | 3587→2441 | 外部冲击，12月，跌幅32% |

注意事项：
- 核心样本是2005年后的6轮牛熊，早期数据参考价值有限
- 创业板有独立行情，不能简单套用上证的牛熊划分

### 1.4 跨指标研究的关键教训

| 教训 | 说明 | 来源 |
|------|------|------|
| 复权模式无影响 | bfq/qfq/hfq 差异 <2%，后续只需算一种 | MACD+MA |
| 周线是最佳频率 | 日线噪音多，月线样本少，周线平衡 | MACD+MA |
| 极端值比交叉有用 | 交叉类信号普遍是噪音，极端位置信号才可靠 | MACD+MA+RSI |
| 行业差异真实存在 | 消费股违反均值回归，不能一刀切 | MACD |
| 信号幅度在衰减 | 方向仍稳定，但收益幅度在缩小 | MACD+RSI |
| 趋势 vs 反转指标稳定性不同 | 趋势指标(MACD/MA)跨时代方向稳定；反转指标(RSI)脱离牛熊环境方向不稳定，但同环境内完全稳定 | RSI L3.5 |
| 大盘股 RSI 更可靠 | 均值回归指标天然适合大盘，小盘信号较弱 | RSI L2.5 |
| 个股信号普遍衰减20-30pp | 个股受财报/重组/游资等非系统性因素干扰，所有指标都大幅衰减 | MACD+MA+RSI L4 |
| 背离是唯一跨层级稳定的信号 | 背离检测"动量衰竭"，RSI底背离指数→个股衰减仅29pp（仍>70%） | RSI L4 |
| 个股层面只有极端值+背离有价值 | 交叉类在个股级全部退化为噪音（48~52%） | MACD+MA+RSI L4 |
| 换手率是唯一层级间方向反转的指标 | MACD/MA/RSI 指数→个股是"衰减"，换手率是"反转"（超高换手指数35%卖出，个股59%卖出）。不能用指数结论指导个股操作 | 换手率 L4 |
| 换手率信号时间稳定性最差 | 同一信号在不同时段可完全翻转（暴增72%→53%）。全量回测被杠杆牛拉高，必须做分段验证 | 换手率 L3.5 |
| 报告必须由模型分析撰写 | 脚本只负责统计计算和DB写入，报告由大模型查询数据后分析撰写。脚本生成的报告缺乏归因、分级、决策树等核心价值 | 换手率报告重写 |

### 1.5 L4 工程教训（强制规则）

从 MA/RSI/换手率 L4 执行中提炼的编码规则：

**索引设计**：
- 数据表：`UNIQUE(ts_code, trade_date)`
- 信号表：`UNIQUE(ts_code, trade_date, freq, signal_name)` + `INDEX(signal_type)` + `INDEX(trade_date)`
- 统计表：`UNIQUE(ts_code, freq, adj)` + `INDEX(ts_code)`
- 建表后立刻 `EXPLAIN` 验证，绝不允许全表扫描

**信号检测性能**：
- 禁止在数据量>100行时使用 Python 逐行循环 + numpy 单值函数
- 优先使用 `pd.rolling()`/numpy 向量化
- `_make_signal_lite` 签名统一为 `(idx, signal_type, signal_name, direction)`
- 每个检测函数写完后 profile，>0.1s/6000行必须优化

**批量 I/O**：
- 批量读取：`WHERE ts_code IN (...)` 每批100只，不逐只查DB
- 连接复用：读/写连接各只开一次，贯穿整个批次
- 批量写入：单连接分片 upsert（1000行/片）
- 复权因子批量读取，不逐只查 adj_factor 表

#### **注意：增量更新**（所有脚本强制支持）：

```
compute脚本: --codes, --start-date, --end-date, --freq, --adj, --all
analyze脚本: --codes, --start-date, --end-date, --freq, --save-signals
```
所有研究脚本必须支持增量更新，避免每次全量重算：

**数据计算脚本**：
- 按 `ts_code` + `trade_date` 做 upsert（batch_upsert），只计算新增或变化的数据
- 检测已有数据的最大 `trade_date`，只拉取之后的新数据进行计算
- 支持 `--start-date` / `--ts-code` 参数，可选全量重算或增量追加

**信号检测脚本**：
- 按唯一约束 upsert，重复运行不产生重复记录
- 支持按日期范围增量检测新信号
- 后续收益字段可在信号首次落库后，由收益更新脚本补填

**日志要求**：
- 输出新增/更新/跳过的记录数
- 输出增量范围（从哪个日期开始更新）

逻辑：查询 MAX(trade_date)，只处理新数据。信号检测需往前多取 lookback 长度的历史上下文。

## 二、分层验证框架

```
Layer 1:   上证指数 → 建立信号行为规律基准
Layer 2:   三大指数(上证/深证/创业板) → 验证结论普适性
Layer 2.5: 宽基(50/300/500/1000) → 市值分层验证
Layer 3:   30个申万一级行业 → 行业差异分析
Layer 3.5: 信号时间稳定性验证(分段) → 结论是否会过期
Layer 4:   全市场个股(~5400只) → 大样本统计
Layer 5:   多指标共振 → 信号质量提升
```

### Layer 1：单一基准

**标的**：上证指数 | **前置**：已阅读牛熊周期报告和MACD方法论

1. 定义信号类型 → 检测信号 → 计算后续收益（T+5/10/20/60）
2. 统计胜率（全量/牛市/熊市）+ 牛熊交叉分析 + 子类型细分
3. **产出**：信号有效性初步判断

### Layer 2：多标的验证

**标的**：上证 + 深证 + 创业板 | 复用 L1 脚本 `--codes` 切换

**判断**：三指数胜率差异 <10% 为普适，否则标记"条件性结论"

### Layer 2.5：市值分层

**标的**：上证50 / 沪深300 / 中证500 / 中证1000

验证信号有效性是否随市值变化。MACD 经验：方向一致但幅度递增（小盘弹性更大）。

### Layer 3：行业差异

**标的**：30个申万一级行业 | **核心产出**：不适用通用结论的"例外清单"

### Layer 3.5：时间稳定性

将数据按时间分段，检查信号跨时段是否稳定。

| 维度 | 含义 | 检验方法 |
|------|------|---------|
| 跨时代稳定性 | 信号方向是否不变 | 分段对比胜率方向 |
| 同环境稳定性 | 同一牛熊内信号是否一致 | 近1年 vs 历史同环境 |

经验：趋势指标(MACD/MA)跨时代方向稳定；反转指标(RSI)同环境内稳定；换手率两种都不稳定。

### Layer 4：个股验证

**标的**：全市场 ~5400只 | 汇总到 `stock_{indicator}_signal_stats` 表

按行业/市值分组分析，找出信号最强/最弱的股票特征。

### Layer 5：多指标共振

组合不同指标的有效信号，测试组合后胜率提升。前提：至少两个指标完成 L1-L4。

## 三、信号类型分类

| 信号类型 | 描述 | 研究优先级 | 经验结论 |
|---------|------|----------|---------|
| **极端值** | 指标到达极端位置 | ★★★ | 最有实战价值（>80%胜率） |
| **背离** | 指标与价格方向矛盾 | ★★★ | 极准但稀有，跨层级最稳定 |
| **位置** | 指标处于某个区间 | ★★ | 月线有效，日线/周线弱 |
| **形态** | 指标的形态特征 | ★★ | 待验证 |
| **交叉** | 两条线交叉 | ★ | 普遍是噪音（~50%胜率） |

## 四、分析流程标准化

```
1. 数据准备 → 确认数据源和覆盖范围
2. 信号定义 → 列出所有信号类型和检测规则
3. 信号检测 → 全量数据运行检测，写入信号表
4. 后续收益 → 计算多窗口收益
5. 统计汇总 → 计数/平均收益/胜率 × 牛熊/行业/市值
6. 结论判定 → >65%有效, 45~55%噪音, >90%但稀有辅助参考
7. 写入DB → 脚本只写数据，报告由模型撰写
```

### 后续收益计算标准

| 周期 | 评估窗口 | 主窗口 |
|------|---------|--------|
| 日线 | T+5, T+10, T+20, T+60 | T+20 |
| 周线 | T+2, T+4, T+8, T+13 | T+4 |
| 月线 | T+1, T+3, T+6, T+12 | T+3 |

胜率定义：买入信号→后续涨为胜，卖出信号→后续跌为胜。

## 五、信号落库规范

### 信号表

信号检测结果必须写入DB（MACD除外，历史原因在脚本内现算）。

- **命名**：`{类别}_{指标}_signal`（如 `index_ma_signal`、`sw_turnover_signal`）
- **核心字段**：ts_code, trade_date, freq, signal_type, signal_name, direction, signal_value, 多窗口收益, 牛熊标注
- **唯一约束**：`(ts_code, trade_date, freq, signal_name)`

### 统计表

- **命名**：`stock_{indicator}_signal_stats`
- **固定字段**：ts_code, freq, adj, kline_count, date_start, date_end, created_at, updated_at
- **信号字段**：每种信号 3 个字段 — `{signal}_cnt` / `{signal}_avg_ret` / `{signal}_win_rate`

## 六、报告规范

### 6.1 报告命名

报告放在 `{indicator}/report/` 目录，命名：`{序号}-{类型}-{指标}-layer{N}-{描述}.md`

| 序号 | 层级 | 示例 |
|------|------|------|
| 01 | L1 | `01-index-rsi-layer1-shanghai.md` |
| 02 | L2 | `02-index-rsi-layer2-comparison.md` |
| 03 | L2.5 | `03-index-turnover-layer2.5-broad-market.md` |
| 04 | L3 | `04-industry-turnover-layer3.md` |
| 05 | L3.5 | `05-time-stability-layer3.5.md` |
| 06 | L4 | `06-stock-turnover-layer4.md` |
| 末尾 | 综合 | `07-methodology-synthesis.md` |

### 6.2 报告生成流程（强制）

**报告必须由大模型基于统计数据分析撰写，禁止由 Python 脚本自动生成。**

```
脚本职责: 信号检测 → 写入DB / 输出日志。禁止生成报告文件。
模型职责: 查询DB数据 → 深度分析 → 撰写报告（含归因/分级/决策树/策略）
```

脚本中不要有报告模板、markdown 拼接等代码。已有的必须清理删除。

### 6.3 报告质量原则

1. **有数据必有解读** — 不能让读者自己猜
2. **有结论必有归因** — 回答"为什么"和"怎么用"
3. **有对比必有判断** — 落到明确的有效/无效/条件性有效判定
4. **有信号必有分级** — S/A/B/C/D级 + 决策树 + 实操策略

### 6.4 报告内容结构（强制）

报告核心论点是：**哪些是买入信号、哪些是卖出信号、哪些是无效信号，强度多大，为什么**。

#### 信号有效性分析框架

- `|胜率 - 50%|` = 信号强度（偏离度），偏离越大信号越有效
- 胜率 > 50%：信号方向与标注一致
- 胜率 < 50%：信号方向与标注**相反**（标注"买入"实际是卖出信号）
- 胜率 ≈ 50%（偏离 < 5pp）：无效信号/噪音
- 偏离 ≥ 20pp 为强信号，10-20pp 中等，5-10pp 弱信号

#### 单指标报告结构

按 **周期 → 牛熊环境 → 信号分类** 三层组织，不要按信号技术分类（乖离率/突破/金叉）罗列：

```
一、日线级别
  1.1 牛市特征
    - 最优入场时机排名（按收益排序，不只是分买入/卖出）
    - 各信号类别分析（乖离率/突破/金叉/排列/支撑阻力在该环境下的表现）
    - 有无有效的反方向信号
  1.2 熊市特征
    - 最优避险信号排名（按可靠性+时效性排序）
    - 同上分析

二、周线级别（同上结构）
三、月线级别（同上结构）

四、历史顶底特征分析（核心章节）
  - 从已确认的历史顶部/底部逆向提取指标状态
  - 提炼顶底共性模式（必要条件 + 辅助条件）
  - 参考 MACD 报告第四章的做法

五、牛熊转换识别
  - 从顶底特征中提炼可操作的转换信号
  - 与其他指标（如 MACD 背离）的交叉验证组合
  - 给出转换信号清单：条件 → 操作建议

六、增益策略
  - 多周期共振（月线定方向→周线定节奏→日线选入场）
  - 多指标组合（MA+MACD等互补策略）
  - 信号分级仓位建议（S/A/B/C级对应的仓位比例）
```

#### 避免的错误

| 错误 | 正确做法 |
|------|---------|
| 结论是"牛市买、熊市卖" | 回答"牛市中哪个信号入场收益最高"、"熊市中哪个信号最早预警" |
| 简单罗列数据表格 | 按收益/可靠性排序 + 归因分析 + 风险调整对比 |
| 只有统计没有解释 | 每个模式都要回答"为什么会这样" |
| 忽略历史顶底逆向分析 | 必须有顶底特征提取章节（参考 MACD 报告） |
| 信号独立分析无组合 | 必须有多指标/多周期组合建议 |
| 把接近50%的胜率当"失效" | <50%是反向信号，要分析反向含义和使用方法 |

### 资金流向研究结论（2026-03-16，已终止）

基于近2年全市场数据（241万条个股 + 5.7万条行业），验证 7 个假设，结论：**Tushare 资金流数据无选股 alpha**。

| 因子 | 20d多空差 | 方向 | 判定 |
|------|----------|------|------|
| 大单净流入 | -0.28% | 反转 | 无效（流入越多越跌） |
| 小单净流入 | +0.27% | 弱正向 | 无效（覆盖不了交易成本） |
| 全口径净流入 | -0.41% | 反转 | 无效 |
| MOD修正大单 | -0.40% | 反转更强 | 无效（修正后更差） |
| 行业拐点(N5) | 57.8%胜率 | 正向 | 勉强可参考，但不可独立使用 |

**根因**：Tushare 的大小单划分基于成交金额阈值（4万/20万/100万），是对"已成交"订单的事后统计，本质上是价格的滞后确认而非前瞻信号。券商研报（华泰2018、开源2022）也指出需要逐笔数据+重定标才能提取有效 alpha，日频汇总数据粒度不够。
