# OE（欧易/OKX）量化交易实操指南
**创建时间**: 20260323 21:00

## 一、什么是 OE 量化交易

OE 即欧易（OKX），全球前三的加密货币交易所。量化交易就是用程序代替人工，根据预设策略自动买卖加密货币。

### 为什么选 OKX 做量化

| 优势 | 说明 |
|------|------|
| **API 完善** | REST + WebSocket，ccxt 完整支持，文档清晰 |
| **费率低** | 现货挂单 0.08%、吃单 0.1%，VIP 更低 |
| **沙盒环境** | 提供测试网（demo trading），零风险验证策略 |
| **衍生品强** | 除现货外，支持合约、期权、跟单，扩展空间大 |
| **流动性** | BTC/ETH 等主流币深度好，大单不易滑点 |

### 量化 vs 手动交易

| 维度 | 手动交易 | 量化交易 |
|------|---------|---------|
| 情绪 | 受恐惧/贪婪影响 | 严格执行策略 |
| 速度 | 秒级反应 | 毫秒级反应 |
| 持续性 | 需要盯盘 | 7×24 自动运行 |
| 纪律 | 经常违反止损 | 机械执行止损止盈 |
| 覆盖面 | 同时看 2-3 个币 | 可监控数十个交易对 |

## 二、我们的系统架构

当前项目已搭建完整的量化交易框架，路径 `crypto/scripts/`，10 个 Python 模块：

```

│                    run_bot.py（主入口）                   │
│         定时调度 → 获取数据 → 计算信号 → 执行交易          │
├─────────────┬──────────────┬────────────┬───────────────┤
│ data_fetcher│  strategy    │  trader    │ backtest      │
│ K线数据获取  │  策略计算     │ 交易执行    │ 历史回测       │
├─────────────┼──────────────┼────────────┼───────────────┤
│ exchange_   │ indicators   │ models     │ database      │
│ client      │ 技术指标      │ 数据模型    │ MySQL连接     │
│ ccxt封装    │              │            │               │
├─────────────┴──────────────┴────────────┴───────────────┤
│                     config.py（配置中心）                  │
│           交易所API / 数据库 / 策略参数 / 风控参数           │
└─────────────────────────────────────────────────────────┘
```

### 数据流

```
OKX API  ──ccxt──→  data_fetcher  ──→  MySQL(stock_crypto)
                                          │
                                          ↓
                                      strategy  ──→  BUY/SELL/HOLD 信号
                                          │
                                          ↓
                                       trader   ──→  执行交易（模拟/实盘）
                                          │
                                          ↓
                                      MySQL 记录  ──→  信号/交易/持仓
```

## 三、从零开始操作（Step by Step）

### 第一步：注册 OKX 并获取 API Key

1. 注册 OKX 账号（需要科学上网）
2. 完成 KYC 实名认证
3. 进入 **个人中心 → API**，创建 API Key：
   - 权限：勾选 **读取 + 交易**，**不要勾选提币**
   - IP 限制：建议绑定服务器 IP（安全）
   - 记录三个值：`API Key`、`Secret Key`、`Passphrase`

> **安全提醒**：
> - API Key 绝对不能泄露，不要提交到 Git
> - Passphrase 是 OKX 特有的第三层验证
> - 建议先用模拟盘（Demo Trading），不配 API Key 也能跑

### 第二步：配置环境变量

在项目根目录或系统中设置环境变量（推荐 `.env` 文件）：

```bash
# OKX API 配置
OKX_API_KEY=你的API_Key
OKX_API_SECRET=你的Secret_Key
OKX_PASSPHRASE=你的Passphrase

# 指定默认交易所为 OKX
CRYPTO_EXCHANGE=okx

# 代理（国内必须）
HTTP_PROXY=http://localhost:7890
HTTPS_PROXY=http://localhost:7890

# 数据库
MYSQL_HOST=127.0.0.1
MYSQL_PORT=3307
```

### 第三步：拉取历史数据

```bash
cd crypto/scripts

# 拉取 BTC 1小时K线，最近365天
python data_fetcher.py --symbol BTC/USDT --timeframe 1h --days 365 --exchange okx

# 拉取 ETH
python data_fetcher.py --symbol ETH/USDT --timeframe 1h --days 365 --exchange okx

# 拉取 4小时线（适合趋势策略）
python data_fetcher.py --symbol BTC/USDT --timeframe 4h --days 365 --exchange okx
```

数据会存入 MySQL `stock_crypto.crypto_kline` 表，后续回测和交易都从这里读取。

### 第四步：回测验证策略

**先回测，确认策略在历史数据上有效，再考虑实盘**：

```bash
# 双均线策略回测（最简单的入门策略）
python backtest.py --symbol BTC/USDT --timeframe 1h --strategy dual_ma --exchange okx

# MACD 策略
python backtest.py --symbol BTC/USDT --timeframe 1h --strategy macd --exchange okx

# RSI 策略
python backtest.py --symbol BTC/USDT --timeframe 1h --strategy rsi --exchange okx

# 组合策略（多策略投票）
python backtest.py --symbol BTC/USDT --timeframe 1h --strategy composite --exchange okx

# 调整风控参数
python backtest.py --symbol BTC/USDT --strategy dual_ma --stop-loss 0.03 --take-profit 0.08
```

回测报告输出示例：
```
============================================================
  回测报告 - 策略: dual_ma
============================================================
  初始资金:     100,000.00 USDT
  最终权益:     112,345.67 USDT
  总收益率:     +12.35%
  年化收益率:   +25.12%
  买入持有:     +8.50%
  最大回撤:     -15.23%
  夏普比率:     1.234
------------------------------------------------------------
  交易次数:     42
  胜率:         52.4%
  盈亏比:       1.850
  总手续费:     1,234.56 USDT
============================================================
```

**关键指标判断标准**：

| 指标 | 合格线 | 说明 |
|------|--------|------|
| 总收益率 | > 买入持有收益率 | 跑不赢持有就别用策略 |
| 最大回撤 | > -20% | 超过20%回撤心态会崩 |
| 夏普比率 | > 1.0 | 风险调整后收益合理 |
| 胜率 | > 45% | 配合盈亏比看 |
| 盈亏比 | > 1.5 | 每笔赢的 > 每笔亏的 |

### 第五步：模拟交易（Paper Trading）

回测满意后，用模拟模式实时跑一段时间：

```bash
# 模拟交易（不需要 API Key，默认就是模拟模式）
python run_bot.py --symbol BTC/USDT --strategy dual_ma --exchange okx --interval 300

# 多币种 + 组合策略
python run_bot.py --symbol BTC/USDT,ETH/USDT --strategy composite --exchange okx --interval 300
```

- `--interval 300` 表示每 5 分钟检查一次
- 模拟交易使用虚拟的 100,000 USDT
- 所有信号、交易、持仓都会记录到 MySQL
- `Ctrl+C` 优雅停止

**建议至少模拟跑 2 周**，观察：
1. 信号频率是否合理（太频繁 → 手续费吃掉利润）
2. 止损止盈是否被正确触发
3. 收益曲线是否平稳

### 第六步：沙盒交易（可选）

OKX 提供沙盒环境（测试网），用虚拟资金在真实交易所环境测试：

```bash
# 沙盒模式（需要配置 OKX 沙盒 API Key）
python run_bot.py --symbol BTC/USDT --strategy macd --exchange okx --sandbox
```

### 第七步：实盘交易

**确认以下条件全部满足后再上实盘**：

- [ ] 回测收益 > 买入持有，回撤可接受
- [ ] 模拟交易跑满 2 周，结果稳定
- [ ] API Key 已配置，IP 已限制
- [ ] 初始资金准备好（建议小资金起步，如 1000 USDT）
- [ ] 代理网络稳定（断网 = 无法止损 = 风险）

```bash
# 实盘交易（需要输入 YES 确认）
python run_bot.py --symbol BTC/USDT --strategy composite --exchange okx --live
```

## 四、内置策略详解

### 1. 双均线策略（dual_ma） — 入门首选

**原理**：快速均线穿越慢速均线时产生信号

| 信号 | 条件 | 逻辑 |
|------|------|------|
| BUY | MA7 上穿 MA25 | 短期趋势转强（金叉） |
| SELL | MA7 下穿 MA25 | 短期趋势转弱（死叉） |

- **优点**：逻辑简单，趋势行情表现好
- **缺点**：震荡市频繁假信号
- **适用**：BTC/ETH 等趋势性强的主流币，1h/4h 周期

### 2. RSI 策略（rsi） — 超买超卖

**原理**：RSI 衡量价格涨跌力度，在极端区域反转

| 信号 | 条件 | 逻辑 |
|------|------|------|
| BUY | RSI < 30 | 超卖，价格可能反弹 |
| SELL | RSI > 70 | 超买，价格可能回落 |

- **优点**：震荡市表现好
- **缺点**：单边行情会反复打脸（强势可以一直超买）
- **适用**：震荡区间内的币种

### 3. MACD 策略（macd） — 趋势跟踪

**原理**：DIF 和 DEA 的交叉 + MACD 柱状图翻转

| 信号 | 条件 | 逻辑 |
|------|------|------|
| BUY | DIF 上穿 DEA 或柱翻正 | 动能转多 |
| SELL | DIF 下穿 DEA 或柱翻负 | 动能转空 |

- **优点**：能捕捉中长期趋势
- **缺点**：信号滞后，入场偏晚
- **适用**：趋势明确的行情，4h/1d 周期效果更好

### 4. 布林带策略（bollinger） — 区间交易

**原理**：价格触及布林带边界 + RSI 确认

| 信号 | 条件 | 逻辑 |
|------|------|------|
| BUY | 价格 ≤ 下轨 且 RSI < 40 | 超跌到支撑 |
| SELL | 价格 ≥ 上轨 且 RSI > 60 | 超涨到阻力 |

- **优点**：双重确认减少假信号
- **缺点**：趋势突破时会反向操作
- **适用**：宽幅震荡行情

### 5. 组合策略（composite） — 推荐使用

**原理**：双均线 + RSI + MACD 三策略投票，2/3 一致才出信号

- **优点**：过滤大量假信号，信号质量高
- **缺点**：信号较少，可能错过快速行情
- **适用**：稳健型交易，追求高胜率而非高频率

**推荐**：新手用 `composite`，有经验后可以针对不同市场状态切换策略。

## 五、风控体系

### 核心风控参数

| 参数 | 默认值 | 说明 | 建议范围 |
|------|--------|------|---------|
| `MAX_POSITION_RATIO` | 30% | 单笔最大仓位占总资金比例 | 10%~30% |
| `STOP_LOSS_PCT` | 5% | 止损比例 | 3%~8% |
| `TAKE_PROFIT_PCT` | 10% | 止盈比例 | 8%~20% |
| `MAX_POSITIONS` | 3 | 最大同时持仓数 | 1~5 |
| `commission` | 0.1% | 手续费率 | OKX 现货 0.08%~0.1% |

### 风控规则

1. **仓位控制**：每笔交易最多用可用资金的 30%，永远不要 All-in
2. **止损必设**：5% 止损是铁律，加密货币一天跌 10%~20% 很常见
3. **止盈跟踪**：默认 10% 止盈，大趋势行情可以改为移动止盈
4. **分散持仓**：最多同时持 3 个仓位，分散风险
5. **手续费意识**：0.1% 看似不多，频繁交易（日均 5 笔以上）一个月手续费可能超过利润

### 资金管理建议

| 阶段 | 资金 | 说明 |
|------|------|------|
| 模拟期 | 0 | 用虚拟资金验证策略 |
| 试水期 | 500~1000 USDT | 小资金实盘感受真实市场 |
| 稳定期 | 3000~10000 USDT | 策略稳定后逐步加仓 |
| 成熟期 | 按承受能力 | 亏完不影响生活的钱 |

## 六、部署方案

### 方案一：本地运行（简单但不稳定）

```bash

  --strategy composite --exchange okx --live \
  --interval 300 > /var/log/crypto_bot.log 2>&1 &

screen -S crypto_bot
python run_bot.py --symbol BTC/USDT --strategy composite --exchange okx --live
# Ctrl+A+D 分离
```

后续可将系统容器化，好处：
- 环境隔离
- 自动重启
- 日志集中管理
- 一键部署/迁移

## 七、OKX 特有注意事项

### 1. API 频率限制

OKX 对 API 调用有速率限制：

| 接口类型 | 限制 | 应对 |
|---------|------|------|
| 公共行情 | 20次/2秒 | ccxt 内置 rateLimit 自动控制 |
| 交易下单 | 60次/2秒 | 不会触及（策略信号频率远低于此） |
| 账户查询 | 10次/2秒 | 注意不要循环查余额 |

### 2. 交易对格式

OKX 的交易对格式与 Binance 略有不同，但 **ccxt 已统一为 `BTC/USDT` 格式**，无需额外处理。

### 3. 最小下单量

每个交易对有最小下单量限制：

| 交易对 | 最小下单量 | 约合 |
|--------|----------|------|
| BTC/USDT | 0.00001 BTC | ~1 USDT |
| ETH/USDT | 0.001 ETH | ~3 USDT |

资金太少（< 100 USDT）可能因为最小下单量限制无法交易。

### 4. 网络问题

国内访问 OKX API 需要代理：
- 系统已配置 `HTTP_PROXY=http://localhost:7890`
- 确保代理服务（Clash/V2Ray 等）稳定运行
- **代理断开 = 无法执行止损 = 最大风险**，务必保证网络稳定

### 5. OKX vs Binance 差异

| 维度 | OKX | Binance |
|------|-----|---------|
| API 认证 | Key + Secret + **Passphrase** | Key + Secret |
| 沙盒 | demo trading（功能完整） | testnet（部分功能） |
| 手续费 | 挂单 0.08% / 吃单 0.1% | 0.1% / 0.1% |
| 合约 | 交割/永续，杠杆最高125x | 永续为主，杠杆最高125x |
| 中文支持 | 好（华人团队） | 一般 |

## 八、进阶方向

当基础系统跑通后，可以往以下方向深入：

### 1. 策略优化
- **参数调优**：用网格搜索找最优 MA 周期、RSI 阈值
- **自适应参数**：根据波动率动态调整止损比例（用 ATR）
- **多周期确认**：1h 信号 + 4h 趋势方向确认
- **量价配合**：加入成交量指标过滤假突破

### 2. 数据增强
- **资金费率**：OKX 永续合约资金费率是很好的情绪指标
- **持仓量（OI）**：合约持仓量变化预示行情转折
- **链上数据**：大户转账、交易所流入流出
- **社交媒体情绪**：Twitter/Reddit 情绪指数

### 3. 交易类型扩展
- **合约交易**：支持做多做空，杠杆放大收益（也放大风险）
- **网格交易**：震荡市自动低买高卖
- **套利交易**：跨交易所价差套利、期现套利

### 4. 系统增强
- **WebSocket 实时推送**：替代轮询，延迟更低
- **监控告警**：异常情况推送 Telegram/飞书通知
- **可视化面板**：权益曲线、持仓分布、信号记录看板

## 九、常见问题

### Q: 量化交易一定能赚钱吗？
**不能。** 量化只是工具，核心是策略。策略不好，再好的系统也亏钱。回测赚钱不等于实盘赚钱（过拟合、滑点、市场变化都会影响）。

### Q: 应该选什么币种？
**BTC 和 ETH 优先。** 流动性好、滑点小、波动规律性相对可预测。山寨币波动大但流动性差，容易滑点严重。

### Q: 用什么K线周期？
- **1h**：信号频率适中，适合大多数策略
- **4h**：趋势策略首选，信号少但质量高
- **15m/5m**：高频策略，手续费压力大，不推荐新手

### Q: 需要多少启动资金？
**模拟交易不需要钱，实盘建议至少 500 USDT 起步。** 太少会受最小下单量限制。

### Q: 如何判断策略失效？
- 连续亏损超过 10 笔
- 回撤超过 20%
- 胜率持续低于 35%
- 收益率持续跑输买入持有

出现以上情况，暂停实盘，重新回测分析原因。
