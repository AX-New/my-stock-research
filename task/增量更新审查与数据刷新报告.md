## 三、增量更新能力审查

### 3.1 审查结果总览

| 脚本 | 主题 | 增量支持 | CLI 参数 |
|------|------|:--------:|----------|
| `rsi/scripts/compute_stock_rsi.py` | RSI 股票 | **支持** | `--start-date`, `--end-date` |
| `rsi/scripts/compute_index_rsi.py` | RSI 指数/申万 | **支持(有BUG)** | `--start-date`, `--end-date` |
| `macd/scripts/compute_stock_macd.py` | MACD 股票 | 不支持 | `--freq`, `--adj`, `--codes` |
| `macd/scripts/compute_index_macd.py` | MACD 指数 | 不支持 | `--freq`, `--codes` |

| `ma/scripts/compute_index_ma.py` | MA 指数/申万 | 不支持 | `--freq`, `--source`, `--codes` |

| `turnover/scripts/compute_stock_turnover_signals.py` | 换手率信号 | 不支持 | `--codes`, `--freq` |

**结论：10 个计算脚本中仅 2 个支持增量更新（均属 RSI），其中 1 个还存在 BUG。**

### 3.2 RSI 增量模式工作原理

```
1. 加载股票全量 K 线数据（保证 EWM/RSI 计算准确性）
2. 计算完整 RSI 序列
3. 过滤: 只取 trade_date >= start_date 的行
4. batch_upsert 写入（仅写增量部分）
```

优点：计算准确，写入量小。缺点：仍需加载全量 K 线，读取开销不变。

### 3.3 发现的 BUG

#### BUG-1: `compute_index_rsi.py` 传递不支持的 conn 参数

```python
# rsi/scripts/compute_index_rsi.py:64
df = load_index_kline(ts_code, freq, conn=conn)  # ← conn 参数不被接受

# macd/scripts/kline_loader.py:182 (实际定义)
def load_index_kline(ts_code: str, freq: str) -> pd.DataFrame:  # ← 无 conn 参数
```

**影响**：RSI 指数计算完全无法运行，所有 12456 个指数代码全部报错。

#### BUG-2: `load_sw_kline` 函数不存在

```python
# rsi/scripts/kline_loader.py:31
load_sw_kline = getattr(_mod, "load_sw_kline", None)  # ← 返回 None

# rsi/scripts/compute_index_rsi.py:128
df = load_sw_kline(ts_code, freq, conn=conn)  # ← NoneType is not callable
```

**影响**：RSI 申万行业计算无法运行。MA 的 `compute_index_ma.py` 也有同样问题（直接 import 会导致 ImportError）。

#### BUG-3: 指数代码数量过多

`get_all_index_codes()` 返回 **12456** 个代码（含 CJ/SH/XI 等各类指数），导致全量计算极其缓慢。实际研究常用的核心指数仅数十个，应增加过滤逻辑或使用 `--codes` 指定。

## 五、改进建议

### 5.1 为所有计算脚本添加增量支持（高优先级）

参照 RSI 的增量模式，为以下脚本添加 `--start-date` / `--end-date` 参数：

1. `macd/scripts/compute_stock_macd.py`
2. `macd/scripts/compute_index_macd.py`
3. `ma/scripts/compute_index_ma.py`
4. `ma/scripts/compute_stock_ma_signals.py`
5. `turnover/scripts/compute_stock_turnover_signals.py`
6. 各主题的信号计算脚本

增量模式设计要点：
- 加载全量 K 线保证计算准确性（EMA/MACD 依赖历史数据）
- 只写入 `>= start_date` 的记录
- 默认全量，传 `--start-date` 时启用增量

### 5.2 修复已知 BUG（高优先级）

1. **修复 conn 参数传递**：`compute_index_rsi.py` 和 `compute_index_ma.py` 中移除 `conn=conn`
2. **实现 `load_sw_kline`**：在 MACD 的 `kline_loader.py` 中添加申万 K 线加载函数
3. **优化指数代码列表**：过滤无效/稀疏指数，或提供常用指数子集

### 5.3 建立统一更新入口（中优先级）

创建 `scripts/update_all.py` 统一入口：
- 自动检测各库最新日期
- 计算需要更新的日期范围
- 按依赖顺序执行（先数据表后信号表）
- 输出汇总报告

## 七、总结

本次审查发现，10个计算脚本中仅2个支持增量更新（均属RSI），且其中1个存在BUG无法运行。通过 RSI 股票级增量更新，成功将 stock_rsi 库的3张日线数据表刷新至20260318，新增65,751条记录，总耗时约12.7分钟。

其余7个脚本（MACD/MA/Turnover）均需全量重算，单次运行需要数小时，不适合日常维护。建议优先为所有计算脚本添加增量支持，并修复RSI指数/申万计算的BUG，以实现高效的日常数据刷新。
